클로드 코드와 함께 이 블로그를 만들면서, 세 가지 언어가 각자 어떤 일을 맡고 있는지 다시 정리해봤다.

## HTML: 구조와 의미

HTML은 페이지에 어떤 내용이 어떤 순서로 존재하는지를 정의한다. `<header>`, `<main>`, `<article>` 같은 태그로 "여기는 헤더고, 여기는 본문이다"라는 의미를 부여했다. 스타일이나 동작이 하나도 없어도, HTML만으로 글의 뼈대는 이미 완성된다.

## CSS: 보여지는 방식

CSS는 그 뼈대를 어떻게 보여줄지 결정한다. 이번 블로그에서는 색상을 직접 하드코딩하지 않고 `--bg`, `--text` 같은 CSS 변수로 묶어뒀는데, 덕분에 다크 모드 전환이 아주 단순해졌다. `data-theme` 속성 하나만 바뀌면 변수 값이 통째로 바뀌면서 색이 전환된다. 미디어 쿼리로는 화면 크기에 따라 여백과 글자 크기를 조정해서 모바일에서도 읽기 편하게 만들 수 있었다.

## JavaScript: 동작과 데이터

JavaScript는 정적인 페이지에 움직임을 더한다. 이 블로그에서는 크게 세 가지 일을 한다.

1. `posts.json`과 `.md` 파일을 `fetch`로 불러온다.
2. 마크다운 텍스트를 파싱해서 HTML로 바꾼다.
3. 다크 모드 버튼 클릭 같은 사용자 동작에 반응한다.

> 세 언어는 서로의 역할을 침범하지 않을 때 가장 깔끔하다. HTML에 스타일을 억지로 끼워 넣거나, CSS로 데이터를 다루려 하면 코드가 금방 지저분해진다.

## 정리

- HTML = 무엇이 있는가
- CSS = 어떻게 보이는가
- JavaScript = 어떻게 동작하는가

프레임워크 없이 셋을 직접 다뤄보니, 평소 프레임워크가 대신 처리해주던 역할들이 더 선명하게 보였다.
