# CLAUDE.md

이 파일은 이 저장소에서 작업할 때 Claude Code(claude.ai/code)를 위한 가이드입니다.

## 프로젝트 개요

마크다운(.md) 파일을 읽어 정적 블로그 웹사이트로 렌더링하는 프로젝트입니다.
프레임워크나 빌드 도구 없이 순수 HTML, CSS, JavaScript만으로 구현합니다
(React, Vue, Next.js, webpack, npm 빌드 파이프라인 등 사용하지 않음).

## 기술 스택 / 제약

- **HTML/CSS/JS만 사용.** 프레임워크, 번들러, 트랜스파일러 도입 금지.
- 브라우저에서 바로 열거나 간단한 정적 서버(`npx serve`, VS Code Live Server 등)로 구동.
- 마크다운 파싱은 외부 라이브러리를 CDN 없이 쓰기 어려우므로, 별도 지시가 없다면 직접 구현한 경량 파서를 사용한다. (CDN 스크립트 사용 여부는 사용자와 먼저 확인할 것)
- 로컬 `.md` 파일을 `fetch`로 읽어오는 구조라면 정적 서버 구동이 필요함 (file:// 프로토콜은 fetch가 막힐 수 있음) — 이 제약을 문서/README에 명시할 것.

## 디자인 원칙

- **네온-온-블랙 단일 테마**: 라이트/다크 토글 없음. 배경은 거의 검은색(`--bg`), 텍스트와 강조 요소는 네온 그린 계열(`--neon`) 단일 톤으로 통일. `css/style.css`의 CSS 커스텀 프로퍼티로 색상을 관리하며, 새 색을 추가할 때도 이 단일 네온 톤 안에서 명도만 조절할 것 (다색 네온으로 확장하지 않음).
- **터미널/모노스페이스 감성**: 사이트 제목·메타 정보·태그·글 경로(`~/posts/파일명`)는 모노스페이스 폰트를 사용해 터미널 느낌을 낸다. 본문 문단은 가독성을 위해 시스템 산세리프 유지.
- **깔끔하고 읽기 좋은 레이아웃**: 넉넉한 줄간격, 적당한 본문 폭(예: max-width ~680-760px), 명확한 타이포그래피 위계.
- **모바일 반응형**: 뷰포트 메타 태그 필수, 작은 화면에서도 여백/폰트 크기/네비게이션이 자연스럽게 조정되도록 미디어 쿼리 사용. 가로 스크롤 발생 금지.
- 불필요한 애니메이션, 무거운 이미지, 외부 폰트 로딩 최소화 — 로딩 속도와 가독성 우선. 네온 글로우(`text-shadow`)나 깜빡이는 커서 같은 테마 연출은 `prefers-reduced-motion`을 존중하고 과하지 않게 유지.

## 구조

```
/
├── index.html            # 블로그 목록 페이지
├── post.html             # 개별 글 렌더링 페이지 (?post=파일명.md 쿼리로 로드)
├── serve.json             # 로컬 프리뷰(npx serve)용 설정 — cleanUrls 비활성화 (쿼리스트링 보존 목적)
├── css/
│   └── style.css          # 네온-온-블랙 테마 토큰, 타이포그래피, 반응형 레이아웃
├── js/
│   ├── markdown.js        # 프론트매터 파싱 + .md → HTML 변환
│   ├── app.js              # 공용 헬퍼(날짜 포맷 등), 모든 페이지 공통
│   ├── list.js              # index.html: posts.json 기반 글 목록 렌더링
│   └── post.js              # post.html: 단일 글 로딩 및 렌더링
└── posts/
    ├── posts.json           # 글 파일명 목록 (새 글 추가 시 여기에 등록)
    └── *.md                  # 프론트매터(title/date/excerpt/tags) + 본문

.claude/launch.json          # `npx serve -l 5500 .` 로컬 프리뷰 서버 설정
```

### 새 글 추가하는 법
1. `posts/`에 `.md` 파일 작성. 상단에 프론트매터 필수:
   ```
   ---
   title: 글 제목
   date: 2026-07-14
   excerpt: 목록에 보일 요약
   tags: 태그1, 태그2
   ---
   ```
2. `posts/posts.json` 배열에 파일명 추가.

### 알아둘 것
- `fetch`로 `.md`/`posts.json`을 읽으므로 반드시 정적 서버로 열어야 함 (`file://` 직접 열기는 CORS로 막힘). `.claude/launch.json`의 `static-server` 설정으로 `npx serve` 구동 가능.
- `serve` 패키지는 기본적으로 cleanUrls가 켜져 있어 `post.html?post=...` 같은 쿼리스트링을 301 리다이렉트 과정에서 날려버림 → `serve.json`으로 비활성화해둔 상태. 다른 정적 서버/호스팅으로 바꿀 경우 이 동작을 다시 확인할 것.
- 마크다운 파서(`js/markdown.js`)는 자체 구현한 경량 파서로, 표(table)나 중첩 목록 등 일부 GFM 문법은 지원하지 않음. 필요해지면 확장.

## 작업 시 유의사항

- 기능 요구사항 이상으로 과도한 추상화(플러그인 시스템, 설정 프레임워크 등)를 넣지 않는다.
- 코드 주석은 최소화하고, 자명하지 않은 이유(WHY)가 있을 때만 작성한다.
- UI 변경 시 실제 브라우저에서 데스크톱/모바일 뷰를 모두 확인한다.
