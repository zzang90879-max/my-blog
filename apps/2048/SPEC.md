# 2048 게임 웹앱 - SPEC

## 개요
- 이름: 2048
- 설명: 방향키/스와이프로 숫자 타일을 밀어 같은 숫자끼리 합치는 퍼즐 게임
- 위치: /apps/2048/ (블로그의 다른 파일은 건드리지 않음, 완전 자체 완결)
- 라이브러리: 없음 (순수 HTML/CSS/JS, CDN도 불필요)

## 파일 구조
```
apps/2048/
  index.html   # 마크업 (헤더, 스코어보드, 4x4 보드, 오버레이)
  style.css    # 스타일 (반응형, 모바일 대응)
  script.js    # 게임 로직 + 렌더링 + 입력 처리
```
- 파일명은 저장소 컨벤션(index.html 등 소문자)을 따름.
- 블로그 css/js와 이름 충돌 방지를 위해 apps/2048/ 내부 파일만 참조 (블로그의 css/style.css, js/*.js는 import하지 않음).

## 게임 로직 (script.js)
- 상태: `board`(4x4 2차원 배열, 0=빈칸), `score`(현재 점수), `best`(최고 점수), `isGameOver`, `hasWon`, `keepPlayingAfterWin`.
- 초기화: 빈 보드 생성 후 랜덤 빈 칸 2곳에 타일 생성(90% 확률 2, 10% 확률 4).
- 이동 처리 공통 함수: 4방향(상/하/좌/우) 입력을 모두 "왼쪽으로 밀기" 로직 하나로 통일 처리.
  - 방법: 방향에 따라 보드를 회전/전치(transpose) → 각 행에 대해 "왼쪽 밀기+병합" 적용 → 역회전.
  - 각 행 처리: 0 제거(compact) → 인접한 같은 값 타일 병합(한 번의 이동에서 타일 하나는 최대 1회만 병합) → 병합된 값을 score에 가산 → 다시 0으로 우측 패딩.
- 이동 후 보드가 실제로 변경되었을 때만 새 타일 1개 랜덤 생성 (변경 없으면 스폰하지 않음 - 불필요한 입력 방지).
- 승리 조건: 병합 결과 타일 값이 2048에 도달하면 `hasWon=true`, "You Win!" 오버레이 표시(계속하기 / 새 게임 선택 가능). 계속하기 선택 시 이후에는 재알림하지 않음.
- 게임오버 판정: 보드에 빈 칸이 없고, 가로/세로 어느 방향으로도 인접한 동일 값 쌍이 없으면 더 이상 이동 불가 → "Game Over" 오버레이 + 최종 점수 표시.
- 새 게임(재시작): 보드/점수/hasWon 초기화, best는 유지.

## 점수판(Scoreboard) UI
- 위치: 타이틀("2048") 아래, 게임 보드 위쪽에 가로로 배치되는 두 개의 박스.
  - SCORE 박스: 현재 점수, 타일 병합 시 즉시 갱신. 점수가 오를 때 살짝 팝업되는 애니메이션(선택, 과하면 생략 가능).
  - BEST 박스: localStorage 키 `2048-best-score`에 저장된 최고 기록. 현재 점수가 best를 초과하면 그 순간 best도 함께 갱신 및 저장.
- "New Game" 버튼을 스코어보드 옆 또는 아래에 배치.
- 게임 규칙 요약 한 줄 텍스트 하단에 표시(예: "방향키 또는 스와이프로 타일을 움직이세요").

## 모바일 대응
- `<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">` 로 핀치줌 방지(스와이프 오작동 방지).
- 보드 컨테이너에 `touch-action: none` 적용해 스와이프 시 페이지 스크롤 발생 방지.
- `touchstart`/`touchend` 좌표 차이로 스와이프 방향 판정 (최소 이동 거리 임계값, 예: 30px 이상일 때만 인식, 더 큰 축을 방향으로 채택).
- 키보드 방향키(desktop)와 스와이프(mobile) 둘 다 동일한 `move(direction)` 함수를 호출하도록 통일.
- 레이아웃: CSS Grid로 4x4 보드 구성, 보드 크기는 `min(90vw, 420px)` 등으로 반응형 지정, 타일 폰트 크기는 `clamp()`로 화면 크기에 따라 자동 조절.
- 버튼(New Game, 오버레이의 재시작 버튼 등)은 터치 타깃 44px 이상 확보.
- 선택 사항(과하면 생략): 화면 하단에 상/하/좌/우 온스크린 버튼 4개 추가해 스와이프가 어려운 환경 보완.

## 외부 라이브러리
- 사용하지 않음. 순수 HTML/CSS/JS로 구현 (CLAUDE.md의 "외부 라이브러리 최소화" 원칙 및 게임 로직의 단순성을 고려할 때 라이브러리 불필요).

## 시각 스타일 참고 (선택)
- 블로그 전역 CSS는 import하지 않지만, 일관성을 위해 비슷한 네온-온-블랙 톤(어두운 배경 + 포인트 컬러)을 apps/2048/style.css 내부에 독자적으로 정의하는 것을 권장. 필수는 아니며 2048 특유의 타일 색상 팔레트(값별 배경색 단계)를 우선해도 무방.

## 구현 범위 밖 (Nice to have, 생략 가능)
- 타일 슬라이드 애니메이션 (transform transition) - 없어도 게임 동작에는 지장 없음, 시간 되면 추가.
- Undo 기능, 다크/라이트 테마 토글 등은 범위 밖.
